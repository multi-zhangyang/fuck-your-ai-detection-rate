# RateAudit 降检诊断技术设计

## 目标

RateAudit 把 FYADR 已有的句法节奏、连接词、模板表达、段落结构和语态诊断从“每个 Diff 块里的零散提示”提升为完整产品闭环：

```text
原文基线 → 分轮结果 → 同规则对比 → 问题热区 → 定向处理 → 再诊断
```

它解决三个问题：

1. 用户在开始前看不到原文有哪些可解释的表达风险。
2. 多轮完成后只能看到当前风险数量，看不到相对原文究竟改善还是退化。
3. 质量页无法把聚合诊断直接落到需要处理的 Diff 段落。

## 非目标与诚信边界

- RateAudit 不是知网、维普、万方或其他第三方 AIGC 检测器。
- 不输出伪造的“AI 率”“通过率”或检测器概率。
- 风险点数只用于同一篇文档、同一套规则下的相对比较，不用于跨论文排名。
- 指标不能驱动碎句、无意义扩写、强塞被动句或删除必要逻辑连接词。
- 事实、术语、数字、引用、否定范围、段落边界和 Word 结构保护继续高于风格优化。

## 数据流

```text
GET /api/rate-audit
  ├─ sourcePath → 原始 TXT / DOCX 可编辑正文
  ├─ document history → 每一轮 outputPath
  ├─ current compare → chunkId / paragraphIndex / outputText
  └─ rate_audit.py
       ├─ stage analysis
       ├─ dimension aggregation
       ├─ baseline/current delta
       ├─ hotspot ranking
       ├─ executable strategy plan
       └─ document edit-contract readiness
```

前端质量页只在用户进入“降检报告”时请求，避免增加工作台首屏和流式进度热路径的负担。

## 五个诊断维度

| 维度 | 关注信号 | 处理原则 |
| --- | --- | --- |
| 句法与节奏 | 连续等长句、表层句模集中、短碎句投机 | 调整重复主语与不合理句界，不设句长配额 |
| 衔接脚手架 | 连接词过密、成组机械推进 | 删除无逻辑作用的连接词，保留必要关系 |
| 模板与空泛表达 | 套话、泛化总结、空泛填充、四字公式 | 回到原文已有对象、动作和关系，不注入新事实 |
| 段落与枚举结构 | 段长过整齐、嵌套编号、冒号—分号模板 | 只做段内优化，不破坏段落和编号 |
| 语态与语域 | 连续同构被动表达 | 语义需要时保留，主体明确时具体陈述 |

## 风险点数

风险点数是可解释的加权计数：

- high：3 点
- medium：2 点
- low：1 点

报告同时返回风险数量、风险点数和各维度点数。`riskPointChange = current - baseline`，负数表示当前规则命中的风险负担下降，正数表示新增或加重。该值不是百分制，也不映射第三方检测概率。

## 热区与建议

当前结果按 `chunkId` 重新执行同一套分析，按风险点数、高风险数量和风险数量排序，最多返回 12 个热区。每条建议包含：

- 维度与优先级；
- 相对原文的趋势；
- 为什么需要处理；
- 不破坏内容合同的处理原则；
- 可直接定位到 Diff 的 `targetChunkIds`。

## 可执行策略与双契约门

RateAudit v2 不再只返回建议列表，还会生成单一 `strategyPlan`：

- 当前没有值得继续处理的信号时返回 `stop`；
- 新增、加重、高等级或局部残留信号返回 `targeted_rerun`，并携带真实 `targetChunkIds` 与推荐提示词；
- 当前维度已经改善且工作流仍有不同职责的下一轮时返回 `next_dimension`；
- 正文范围、标题冻结、模型输入或格式锁不就绪时返回 `blocked`。

报告同时嵌入 `contentContract` 和统一 `readiness`。策略计划只有在正文与格式契约通过时才可能 `canExecute=true`，避免“算法建议继续改写，但标题或格式范围已经失控”的矛盾状态。完整契约见 [`DUAL_CONTRACT_DESIGN.md`](DUAL_CONTRACT_DESIGN.md)。

## 性能和边界

- 单阶段最多分析 300,000 字符；超限时在靠近上限的段落或句子边界截断，并显式返回 `truncated=true`。该上限覆盖常规学位论文，同时避免异常大文本长期占用 Web 线程。
- 风险归类复用阶段指标快照，不对同一文本重复执行整套正则与句长统计。
- 最多展示 12 个热区，避免大论文导致页面过载。
- API 的 `sourcePath` 仍受 `origin/` 路径边界限制；显式 `outputPath` 只能指向 `finish/`，且必须由当前文档历史记录引用，或由 `compare.docId` 证明属于当前源文档，防止跨文档串档。
- 分析完全离线，不调用模型、不消耗 API Token。

## 验证合同

- 纯分析回归：高模板原文相对自然结果必须显示风险下降；退化结果必须显示风险上升。
- API 回归：路径边界、基线模式、历史阶段、compare 热区、策略决策和正文契约结构稳定。
- 前端回归：服务接口、类型、加载/错误/空态、轨迹、维度、双契约门和 Diff 定位保持连通。
- 总闸：前端类型检查、构建、Python compile、核心回归和浏览器冒烟均需通过。
