# Changelog

本文件记录 FYADR 的用户可见变更。项目在 `0.x` 阶段采用轻量版本记录，稳定后再切换到完整语义化版本。

## Unreleased

### Added

- 新增七个 URL 驱动的工作区：工作台、降检报告、模型配置、提示词、保护区地图、历史记录和启动诊断；支持深链、刷新恢复与浏览器前进/后退。
- 提示词工作区新增流程模板编辑器，可维护名称、说明、默认提示词顺序、默认编排上限和运行轮次上限。
- 新增 RateAudit 相对质量诊断、段落定位与工作台 Diff 联动。
- 增强历史恢复、checkpoint、导出资产治理、历史库备份与修复能力。
- 提示词编辑器新增历史版本面板，可核对备份时间、大小、行数与正文后确认恢复。
- 历史维护新增当前库完整校验、备份有效性状态和异常备份恢复阻断；启动诊断展示历史库规模、空间与自动治理结果。
- DOCX 导出增加正文契约、保护区证据、OOXML 结构审计和阻断式格式锁。
- 增加桌面折叠导航、移动抽屉、键盘导航、焦点恢复和无障碍状态播报。
- 新增可选的单用户登录、HttpOnly 会话、CSRF 校验和失败登录限流；未配置密码时保持本地免登录模式。
- 长文档 Diff 超过 40 块时启用动态高度窗口化，并保留筛选、焦点定位、审阅和局部重跑交互。

### Changed

- 产品品牌统一为 **FYADR / Fuck your AI detection rate**，界面副标题统一为“论文 AI 降检平台”。
- 模型配置支持 OpenAI-compatible Chat Completions 与 Responses、多服务商、按轮路由、流式响应、重试和 `1–16` 并发。
- Web UI 默认改写并发调整为 `2`；实际吞吐仍由服务商限流和运行环境决定。
- README、部署说明、贡献指南和发布检查改为面向公开用户的安装、数据流与安全说明。
- Docker 将提示词内容、注册表、流程模板和自定义提示词作为完整持久状态保存，容器重建不再丢失编辑。
- Docker 会将旧版 `data/prompts-custom/` 中的有效提示词安全迁移到统一目录；冲突内容使用稳定别名保留，删除后不会在重启时重新出现。
- 开发回归改用跨 Node.js 20/22 的 TypeScript 执行器。
- 导出接口只接受 POST，移除会创建文件的旧 GET 兼容入口。
- Python 运行/开发依赖改为带哈希的可复现锁文件；CI 增加真实生产容器构建、非 root 运行和挂载目录写入烟测。
- 高频进度改为有界事件历史、可续传 SSE 与节流落盘；轮次断点改为线性追加 journal，避免长文档反复重写完整状态。
- 浏览器文档上传改为 multipart，服务端流式计数与计算内容哈希；SQLite 新备份改为 gzip，并继续兼容旧原始备份。

### Fixed

- 修复快速 Back/Forward 时旧的脏稿确认覆盖新导航状态的问题。
- 修复提示词读取、保存、删除和流程切换的并发覆盖与错误归因问题。
- 修复历史列表、资产、孤儿扫描、维护、备份、诊断和文档恢复的陈旧响应覆盖问题。
- 修复手动历史文档快速切换时旧状态、快照、模型路线或反馈覆盖新选择的问题。
- 修复流程模板允许 4–12 轮默认编排、但工作台仍只显示 1–3 轮的问题。
- 修复移动端侧边栏 rail 重复显示、抽屉关闭焦点和脏稿确认顺序问题。
- 修复生产容器 CI 无法识别根相对 Vite 资源、`docker top` 缺少 PID 列以及完整回归超时预算不足的问题。
- 修复 Windows 下 SQLite、DOCX 测试临时句柄未关闭的问题，并将哈希依赖锁改为 Dependabot 可解析的标准文件名。

### Removed

- 移除学校规范解析、学校模板套版、学校规范对照页面及相关格式写回路径；旧 `school_rules` 配置只作为兼容输入迁移到 `preserve_original`。
- 移除外部 AIGC 检测报告上传、解析、匹配和自动重跑链路。

### Security

- 已保存 API Key 与 Base URL 绑定；修改地址后必须重新输入凭据。
- POSIX 配置目录和文件使用 `0700/0600` 权限并通过原子替换写入。
- POSIX 服务使用 `umask 077`，并在启动时收紧文档、历史、任务状态、提示词和备份的旧权限，同时保留只读锚点语义。
- DOCX 上传在持久化前限制 ZIP 部件数、单项/总解压大小与压缩比，拒绝路径穿越、重复部件、加密/符号链接部件、DTD/实体和缺失的 OOXML 核心结构。
- Docker 默认仅发布到 loopback；共享部署可启用内置单用户登录，并仍需 HTTPS、网络访问控制和网关限流。
- 登录会话使用持久签名密钥、HttpOnly Cookie 和 CSRF header；API 健康探针与 CORS 预检保持公开。

### Known Limits

- FYADR 不提供第三方 AIGC 检测分数，也不保证任何平台的结果。
- 登录只提供单用户入口保护，不包含多租户隔离或细粒度授权，不适合直接暴露后端端口到公网。
- DOCX 格式保护不代表二进制字节相同；正文长度变化仍可能影响换行、分页和页码。

## 0.1.0 - Initial public preview

### Added

- Web 工作台：上传 DOCX、选择改写策略、查看实时 Diff、导出 Word。
- 段落级改写管线：按自然段处理正文，保护标题、目录、图表、公式、参考文献等非正文区域。
- 多轮策略编排：支持自定义轮次、prompt 组合、追加轮次和模型服务商配置。
- 模型服务商仓库：支持默认配置、按轮次选择服务商与模型、请求频率限制和连接测试。
- 任务恢复能力：支持断点记录、失败后续跑、历史记录切换和运行状态恢复。
- 学校规范解析：支持默认格式规则、AI 结构化解析、兜底抽取和导出样式审计。
- DOCX 导出保护：保护区内容不交给模型改写，但允许按学校规范套用样式。
- 开源安全审计：提供本地审计脚本，拦截 API Key、私有模型地址、个人路径、旧项目名和乱码文本。

### Changed

- 项目名称统一为 `FYADR`。
- Prompt 文件改为 `fyadr-*` 命名，保留核心中文改写 prompt 的稳定边界。
- 前端布局以 Diff 区域为主视觉，左侧导航支持折叠，通知与诊断信息做了收口。
- 历史记录操作区分记录、中间产物、导出文件和源文档，避免误删语义混乱。

### Removed

- 移除外部 AI 检测报告反馈链路：不再提供 SpeedAI / PaperPass PDF 上传、解析、匹配和按报告自动重跑入口。
- 移除与当前 Web 工作流无关的旧技能入口。
- 移除独立英文摘要处理方案，英文段落回到与普通正文一致的改写流程，仅保留语言输出校验。
- 移除谨慎整篇轻扰动功能，避免复杂流程导致误判和卡死。

### Known limits

- 学校规范解析依赖输入说明质量和模型结构化输出能力；系统提供本地兜底，但仍建议导出后人工审阅格式。
